Formatting

Until now we have been concatenating strings using the addition operator. While this works for simple strings, it does not scale well. We will go over a few different options for formatting, so when the time comes in your programs, you can decide for yourself what the best solution is.

The first way to handle strings is with %. This is the way many programs written in Python 2 do it.

We have three variables: dog_name, owner_name, and dog_age. We use %-formatting to build a sentence using those three variables.

Python looks for % signs in a string, followed by a letter, which could be s, d, or f. These stand for string, decimal, and float, respectively. At the end of the string, we include a % sign followed by a tuple of the variables that Python should use to substitute the escape sequences used within the string.

I purposely excluded floats in the example above because they can have unexpected results.

Wait that’s not right — it should have only shown two decimal places, not six. What happened?

By default, %f shows six decimal places. We can override this by using %.2f instead of %f. The . is the decimal point and 2 is the number of decimals to show after the decimal.

If you put the 2 in front of the decimal, you will get enough spaces in front of the whole number from the $.

We can also make this a bit cleaner by using string alignment.

Now we pad the first string with ten characters by using %-10s. Spaces are added around Vanilla and Chocolate, so the strings are as long as strawberry. If we did not include the - sign, we would add enough spaces to the front of the string so that Vanilla and Chocolate would be right-aligned with Strawberry.

The next string formatter is .format.

.format is the “new” way to format strings and makes formatting strings easier to read.

Instead of searching for % signs as it did previously, it replaces {} with the variable passed in the format function. In this example, each placeholder is replaced in the same order as the variables appear in .format.

If you have multiple placeholders that should be filled in with the same variable, you can place numbers in the curly braces that signify the position of the arguments in .format. e.g. {0}, {1}, {2}. Anywhere a variable is used more than once, you reuse its position in curly braces.

If you need to do any formatting of float values, you can use the following syntax to get the same results as above: {0:2.2f}. The colon separates the variable from the formatting. 2.2f puts up to two spaces before the decimal point, and up to two zeroes after the decimal point.

You can also name the positions and provide them passed in as names in any order.

.format also has one more trick up its sleeve but only if you use named positions.

**locals() is a handler (or a pointer to a pointer) that looks at all of the variables that are available at the time it is executed. It scans the named positions and looks for variables that have the same name. If it finds one, it substitutes the named position for the variable found. If not, you receive a KeyError. This helps keep your code simple.

There are more — I could write this entire article on formatting — but now we are going to go over one more type of formatter, before we move on to list comprehensions.

There are many types of escape sequences, \t makes tabs, \s is a space, \\ makes a backslash, \" makes a double quote and \' creates a single quote. Both quotes are only necessary when you use a double or single quote within a string wrapped in the same quote type.

To ignore these, we can use raw strings.

"This is how you make a new line: \\n"
# can also be typed as
r"This is how you make a new line: \n"

The r before the first quote tells Python to ignore escape sequences. This is useful for when you are making regex patterns.

What I originally wanted to go over was f in front of strings. f takes away the need to add .format at the end of your string and looks for local variables to insert into the string.

Lambdas

Not that type of lamb!

Lambdas are anonymous functions. That is to say they do not have a function signature. Lambdas can have any number of arguments but always perform the same expression:

When you create a lambda, you can pass it around like a variable; however, it is meant to be used as an argument to higher-order functions. Higher-order functions take a function as one of their arguments or return a function as a result.

Here are a couple of examples of higher-order functions and a description of what they do:

  • filter(func,x) — filters elements that meet an expression from a list
  • map(func,x) — performs an expression on each item in a list

Lambdas are tricky, and while they save you the amount of typing you might have to do otherwise, they can make your code hard to understand. Try to ensure that your code is descriptive and explicit when using lambdas.
