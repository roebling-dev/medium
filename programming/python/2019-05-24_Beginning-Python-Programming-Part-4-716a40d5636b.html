Decision Making

Every day we make decisions. When we get up in the morning we might get a cup of coffee, but for me getting a cup of coffee requires decisions on whether or not I make a cup. First I need to check if I have coffee, second, I need to check if I have sugar, next I check to make sure I have milk, last I check if I have a clean coffee mug.

If the answer to any of those is no, either I don’t do anything, or I do what is needed to get a cup of coffee.

The same is true in programming and we can make these decisions using if statements.

If statements look like this:

We have two boolean variables at the top: has_coffee and made_cup. We use the syntax if has_coffee to check the value of has_coffee. If coffee evaluates to true, then we perform the code below indented by 4 spaces underneath. If has_coffee was false, we’d skip over everything indented below and the value of made_cup would still be false.

Because Python allows us to use strongly typed variables, I could have used has_coffee: bool = True instead to that denote has_coffee should be a boolean value. It doesn’t mean we can’t override that later on with int, float, list, or dict later, it just helps us keep track of what value should be used.

I also used a linking verb in the variable name to help me remember this is a boolean value. A nicety that I picked up from Apple while learning Swift.

Any time I make booleans I always try to use something like is_sunny, did_rain, or user_is_at_least_thirteen. Regardless of what I use, it should state the purpose of the variable so someone else looking at your code has no doubt on how to use it. Sometimes contextual clues like this help you remember other variables that were set earlier on in the program.

Now I’m going to give you a little more power and the ability to make better decisions, because if statements also have another piece that lets you do something if the original if evaluation resulted to false.

Else clauses allow you to do something if the initial evaluation resulted to false. In this case, has_coffee is set to False and made_cup isn’t yet created.

As we flow through the if statement, we first check if has_coffee. Since we don’t have coffee, we skip down to the else clause and set made_cup to False. Some developers will add in elif, short for else if, statements in between for any other cases. Here’s how that looks:

Because we hit a branch we were looking for, we stop checking the age variable and return from the if statement. In other words, the else statement is never even seen.

There is a tool that we use called a truth table. Its purpose is to show the possible results of what could happen in a scenario.

From the truth tables above, we see the AND operator and the OR operator. AND is used to check for TRUE in all variables. If we were to add a third column named C we would extend out the table to include all variations between A, B, and C. If one value is FALSE, then the result will be FALSE. It’s not a democracy or a weighted average. Loops

There are two different types of loops we will talk about in this post. The first is a while loop, the second is a for loop.

While Loops

Let’s think about this as a scenario, “While I walk the dog, I’m going to…”, or “While I’m on my lunch break, I need to…”. This is kind of how while loops work. While something is true, do this.

While loops are easy to spot. You start with while, have some condition to check, and in the body of the while statement, you have your code. What do you think would happen if I left out count += 1?

That’s right, the code would go on indefinitely. This is bad. So bad it is one of the quickest ways to find your self back on the job market. No one likes infinite loops.

You can try it out on your own computer—just use CTRL+C to break out of the loop. Some things like web servers rely on infinite loops to process requests. Yes, I know I just told you don’t use them, but in event-driven applications like web servers, you process each request one at a time and need to reset to process the next request. (Think of it like an escalator.)

The reason I said they are bad is that when your while loop is not event-driven, it just keeps going and going and going and… this burns up CPU and can slow down or halt the computer from performing other work.

For Loops

The sibling to while loops are for loops. For loops are used primarily in lists and dictionaries, but they can also be used in a range of characters or numbers. Here’s how for loops look:

OK, so a few new things here. First of all, let’s talk about the for loop.

A for loop needs a few things to get started. First, you need to give it a variable that you plan to use in the body of the loop and you should give it a meaningful name. In the first example, we use for name in names. This reads for the current name we are iterating over in the list names.

Next, we add logic on what to do with the name. In our case, we create a nice long string with all of the names using string concatenation.

When we have looped through all of the names, the for loop exits.

Dictionaries are a little different but are based around the same idea. In our example above, we have some weird syntax up front, for (name, type). This actually introduces another type we haven’t talked about called a tuple.

A tuple is a list of comma-separated variables surrounded in parentheses that can contain a value for each variable in the list. They are passed around as a single variable. We will cover them more in the future, but for now, that’s all you need to know.

The reason for this is dictionaries have two parts to each item they contain. A key and a value. The key is the first part of the dictionary, the value is the second part. Keys are used for accessing values. If you were using a real dictionary, the word that you want to look up would be your key, and the definition you find would be the value.

When we use for (name, type) it can be directly translated to for (key, value) in dictionary. So with that said let us move onward. We iterate through each key-value pair in the dictionary, for every key, or name, we append it to pet_names. For every value, or type, we check to see if we have a key for the type. In the case pet_type_counts keys would be “Dog,” “Cat,” “Mouse,” or “Cricket.” Because dictionary keys may or may not exist, we need to be careful. First, we check if pet_type_counts[name] is None. If it is we need to set the initial value to 1 since our current iteration holds that type. Otherwise, we do have a value in pet_type_counts[name] and we just need to update the value. If you try to pull a value out of a dictionary where the key doesn’t exist, you get a KeyValue error and your program will crash. I promise I will talk about errors and how to deal with them in a future article.
