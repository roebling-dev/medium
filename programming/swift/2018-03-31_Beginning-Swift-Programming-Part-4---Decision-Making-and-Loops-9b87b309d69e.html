Decision Making

Every day we make decisions. When we get up in the morning we might get a cup of coffee, but for me getting a cup of coffee requires decisions on whether or not I make a cup. First I need to check if I have coffee, second, I need to check if I have sugar, next I check to make sure I have milk, last I check if I have a clean coffee mug.

If the answer to any of those is no, either I don’t do anything, or I do what is needed to get a cup of coffee.

The same is true in programming and we can make these decisions using if statements.

If statements look like this:

Here I have two boolean variables at the top, one for hasCoffee and one for madeCup. I know that I have coffee in my kitchen, but I haven’t made a cup yet, so we set hasCoffee to true, and madeCup to false.

Next is the actual if statement. We use the syntax if hasCoffee to check the value of hasCoffee, if coffee evaluates to true, then we perform the code in the curly braces {...}. If hasCoffee was false, we’d skip over everything in the curly braces and the value of madeCup would still be false.

Apple’s coding convention is to use a linking verb before booleans so we know later on what type it is without having to refer back to its type.

hasCoffee uses Apple’s coding convention, but we could also say isSunny or didRain where it makes sense. An age test might use userIsAtLeastThirteen, this not only tells you that you should expect true or false back, but it also helps you think about how the value may have been set. Sometimes contextual clues like this help you remember other variables that were set earlier on in the program.

Now I’m going to give you a little more power, and the ability to make better decisions, because if statements also have another piece that lets you do something if the original if evaluation resulted to false.

Else clauses allow you to do something if the initial evaluation resulted to false. In this case hasCoffee is set to false and madeCup isn’t set at all, but it will be a boolean eventually, so we make it a strongly typed Boolean up front without a value.

WARNING: If you try to use a non-optional Boolean variable that has not been instantiated, the compiler will scream at you.

Loops

There are two different types of loops we will talk about in this post. The first is a while loop, the second is a for-in loop.

While Loops

Let’s think about this as a scenario, “while I walk the dog, I’m going to…”, or “while I’m on my lunch break, I need to…”. This is about how while loops work. While something is true, do this.

While loops come in two forms while and repeat-while. Let’s look at each now.

The difference between a while loop and a repeat-while loop is that the while loop checks the value before performing logic, repeat-while checks the value after performing logic.

You need to be careful when making loops. You don’t want to end up with an infinite loop, that is a loop that never ends. You may not hang up your computer, at least for very long, but your program would just be busy calculating infinity.

while true means run indefinitely. true is a constant for 1. false is a constant for 0.

if currentNumber % numberToCountBy != 0. If the remainder of 11 / 4 is not 0, add 1 to currentNumber, so we don’t trap ourselves in an infinite loop, and skip over this iteration using continue. If we didn’t finish performing the logic for this round

You can also write multi-line if statements on one line using ; at the end of each statement. if currentNumber % 4 != 0 { currentNumber += 1; continue } NOTE: this decreases readability in most cases.

For-in Loops

The sibling to while loops are for-in loops. For-in loops are used primarily in arrays and dictionaries, but they can also be used in a range of characters or numbers. Here’s how For-in loops look:

Ok so a few new things here, first of all, let’s talk about the for-in loop.

A for-in loop needs a few things to get started. First, you need to give it a variable that you plan to use in the body of the loop. You cannot change the variable, but you can give it a meaningful name for each iteration of the loop. In the first example, we usefor name in names. This reads “for the current name we are iterating over in the array names.

Next, we add logic on what to do with the name, in our case we create a nice long string with all of the names using string concatenation.

When we have looped through all of the names, the for-in loop exits.

Dictionaries are a little different but are based around the same idea. In our example above, we have some weird syntax up front, for (name, type) this actually introduces another type we haven’t talked about called a tuple.

A tuple is a list of comma separated variables surrounded in parenthesis that can contain a value for each variable in the list, they are passed around as a single variable. We will cover them more in the future, but for now, that’s all you need to know.

The reason for this, is dictionaries have two parts to each item they contain. A key and a value. The key is the first part of the dictionary, the value is the second part. Keys are used for accessing values. If you were using a real dictionary the word that you want to look up would be your key, and the definition you find would be the value.

When we use for (name, type) it can be directly translated to for (key, value) in dictionary. So with that said let us move onward. We iterate through each key-value pair in the dictionary, for every key, or name, we append it to petNames. For every value, or type, we check to see if we have a key for the type. In the case petTypeCounts keys would be “Dog”, “Cat”, “Mouse” or “Cricket”. Because dictionary keys may or may not exist, we have to use them as optionals. So first we check if petTypeCounts[name] == nil. If it is we need to set the initial value to 1 since our current iteration holds that type. Otherwise, we do have a value in petTypeCounts[name] and we just need to update the value. Because it’s optional, we can force unwrap the value using ! but be very careful doing this, always make sure the value is not nil beforehand, otherwise ! lives up to its name as the crash operator. Your program will crash if it forcefully unwraps a nil value.

Finally, there are ranges. You can make a range in a few different ways I’ll show them to you now.

  • ... — full-open (1…10) is the same as writing out 1 through 10
  • ..< — half-open less than (1 ..< 10) is the same as writing 1 through 9
  • 2... — one-sided full-open is the same as writing 2, 3, 4, 5, 6,…to infinity
  • ..< 2 — one-sided half-open allows you to check for any value below 2 including -(infinity)

In the last example, I show you how to use a for-in loop with a range. First we give it the variable value to use in the loop, then we give it count so it knows which number to start at, finally, we give it a full-open range of 1...10. The loop iterates through all of the numbers and adds them together.
